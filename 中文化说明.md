# Decky LSFG-VK 中文化版本

## 简介

本项目在原版 `decky-lsfg-vk` 的基础上实现了完整的中文化。

**特点**：
- ✅ 不修改任何源代码
- ✅ 通过构建时替换实现翻译
- ✅ 易于维护和更新
- ✅ 可随时切换回英文或其他语言

## 快速开始

### 构建项目

```bash
# 安装依赖
pnpm install

# 构建（自动应用中文翻译）
pnpm run build

# 开发模式（监听文件变化）
pnpm run watch
```

构建完成后，`dist/` 目录中的文件将包含中文界面。

### 修改翻译

1. 编辑 `translations.js` 文件
2. 添加或修改翻译条目
3. 重新运行 `pnpm run build`

## 文件说明

| 文件 | 说明 |
|------|------|
| `translations.js` | 翻译映射表（英文→中文） |
| `rollup-plugin-i18n.js` | 自动翻译插件 |
| `rollup.config.js` | 构建配置（已集成翻译插件） |
| `i18n-README.md` | 详细的技术文档 |

## 工作原理

1. 在 Rollup 打包过程中，插件会读取 `translations.js`
2. 扫描所有源代码中的字符串
3. 将匹配的英文文本替换为对应的中文
4. 输出中文化的构建文件

**源代码保持英文，只有构建输出是中文！**

## 翻译覆盖

当前已翻译 **123 条**文本，包括：

- 所有按钮和菜单项
- 配置选项和说明
- 状态消息和提示
- 错误和成功消息
- 模态对话框内容

## 切换语言

### 构建英文版

临时重命名或删除 `translations.js`，然后构建：

```bash
# Windows
ren translations.js translations.js.bak
pnpm run build
ren translations.js.bak translations.js

# Linux/Mac  
mv translations.js translations.js.bak
pnpm run build
mv translations.js.bak translations.js
```

### 添加其他语言

1. 创建新的翻译文件，如 `translations-ja.js`（日语）
2. 修改 `rollup.config.js` 中的 `translationsPath`
3. 重新构建

## 常见问题

### Q: 为什么不直接修改源代码？

A: 这样做有几个好处：
- 便于同步上游更新
- 保持代码库整洁
- 可以轻松支持多语言
- 翻译和代码分离，便于维护

### Q: 某些文本没有被翻译？

A: 请检查：
1. `translations.js` 中是否包含该文本
2. 文本是否完全匹配（包括标点、空格）
3. 重新构建项目
4. 查看构建日志中的 `[i18n]` 信息

### Q: 如何贡献翻译？

A: 欢迎提交 Pull Request！
1. 编辑 `translations.js` 添加或改进翻译
2. 测试构建确保无误
3. 提交 PR
